## Overview

This is an open-ended assignment. You are expected to think about and implement a program of your own choosing (e.g., an information system, a personal assistant, etc.). The emphasis is on **creativity** and writing code at a high level, using all the tools and principles acquired throughout the course.

You may use tools learned through self-study, provided you understand them well and can explain any additions that go beyond the in-class material. The goal is to **plan and build a software solution** that combines functions, loops, arrays, structs, dynamic memory allocation, pointers, file handling, and event/input handling - delivering a challenging, creative, and clear user experience.

---

## Technical Requirements

The program must include:

- **At least 2 structs** for data organization
- **At least 8 functions**, including `main`, handling the program's logic
- Use of **arrays**, **dynamic memory allocation**, **files**, and **meaningful function names**
- **Short documentation** (as a comment) for every function
- **Error handling** for invalid input
- **Mandatory split** into `.c` and `.h` files, divided by logical grouping
- **Originality and creativity are required**
- There is no textbook solution - you are the creators of this software

## Grading Emphasis

### Program Presentation

**Program Description**
- Clear description of the program's purpose (needs clear goals and usefulness)
- Clear and detailed explanation of how the program is used
- Examples of different inputs

**Data Structure Description**
- Presentation of structs used and the arrays they work with
- Brief explanation of their role and usage

**Function Description**
- List of central function names and their parameters
- Brief and clear description of each central function's role

**Algorithm Description**
- High-level flowchart / explanation of the main algorithm

**Dynamic Memory Allocation Description**
- Explanation of why dynamic memory allocation is used

**File Usage (if applicable)**
- Explanation of how files are used and their purpose

**Reflection**
- Description of a challenge encountered during development and how it was overcome

---

### Software Project (Code)

**Logical separation**
- Correct use of functions to separate game logic
- Minimum 8 functions total including `main`, for game management, printing, input validation

**Struct usage**
- Correct use of at least two structs (e.g., user representation, data, game board, resource)

**Code readability**
- Clear and meaningful variable, struct, and function names
- Short documentation (comments) for **every** function - mandatory

**Runs correctly without failures**
- Program runs from start to finish without crashes or critical errors
- All features offered by the program work as intended

**Edge case handling**
- Correct handling of boundary states (e.g., attempts exhausted, goal achieved)
- Correct handling of invalid input (e.g., entering illegal values)

**User experience**
- Clear menus for the user (e.g., action options, operating instructions)
- Display of program status after each action (e.g.: program state, resources)
- Ease and convenience of use
- Clear and detailed feedback after each action, in case of an error or illegal selection
